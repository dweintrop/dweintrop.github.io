Comparing Blocks-based and Text-based Programming

Blocks-based programming environments are growing in popularity and are increasingly being used in formal introductory programming contexts. To date, much of the work evaluating such tools has focused on their effectiveness in out-of-school contexts and emphasized engagement and attitudinal measures over content mastery. Given their growing presence in classrooms, it is important to understand the benefits and drawbacks to the use of the blocks-based programming approach in formal learning contexts.

For my dissertation I am investigating the differences between blocks-based, text-based, and hybrid blocks/text introductory programming tools. To understanding the afordances and drawbacks of different programming modalities, I am carrying out a mixed-method, quasi-experimental study in high school computer science classes. The goal of this work is to better understand the tools we are using to introduce todayâ€™s learners to computer science and lay the foundation for creating the tools of tomorrow.

I have written-up a number of versions of my dissertation study, including a 2-page summary, a 4-page summary, and a 73-page formal proposal.

Hybrid Blocks/Text Programming Environments

As part of my dissertation study, I have created two hybrid blocks/text programming environments. Both environments were designed to have 3 distinct modes (a block version, a text version and a hybrid block/text version) and were used by 3 classes of high-school students for 5-weeks. Findings from the use of these tools in the classroom can be found on the publications page.

Pencil.cc

Pencil.cc is my fork of the Pencil Code and Droplet projects. Droplet is a browser-based editor that seamlessly convert programs between blocks-based to text-based forms. Pencil Code is a more fully featured development environment build with Droplet that adds a Logo-esque library and visual executation space for learners to use. My fork of these tools adds new hybrid blocks/text features as well as advanced logging and a number of other UI features to support research.

You can play with Pencil.cc here and see the code here.

Snappier!

In the fall of 2014 I ran a pilot of my blocks/text comparison study using a modified version of the Snap! programming environment. In my modified version students could view JavaScript implementaitons of the graphical scripts they had assembled as well as had the ability to define new custom blocks using JavaScript.
